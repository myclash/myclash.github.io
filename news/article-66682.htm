Error response from daemon: Get https://registry-1.docker.io/v2/ … read: connection refused

首页 / 新闻资讯 / 正文

使用docker拉取镜像时出现下面状况，根据个人理解这个情况应该是docker无法解析https://registry-1.docker.io/v2/这个网站，所以需要修改类似DNS server这样的东西，在这里修改的是resolv.conf文件里面的nameserver

zy@zy:~$ sudo docker pull ubuntu:16.04 Error response from daemon: Get https://registry-1.docker.io/v2/: dial tcp: lookup registry-1.docker.io on [::1]:53: read udp [::1]:58060->[::1]:53: read: connection refused

修改文件/etc/resolv.conf

sudo vim /etc/resolv.conf

将nameserver修改为8.8.8.8或者114.114.114.114

这里提供一个网址https://dns.icoa.cn/，通过这个网址可以查看很多可用的DNS服务器

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8) #     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN # nameserver 127.0.1.1 nameserver 114.114.114.114
